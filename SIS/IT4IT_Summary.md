## 1. Описание продукта

Разработанный инструмент — это веб-приложение на базе Streamlit, которое использует Google Gemini API для автоматического анализа клиентских отзывов. Целевой клиент — малый бизнес в сфере услуг (ресторан, кафе, салон красоты), у которого нет ресурсов нанимать аналитика, но есть реальная потребность понимать, что говорят клиенты.

**Проблема:** Владелец ресторана в среднем тратит 1.5–2 часа в день на ручное чтение отзывов с разных платформ. При этом часто упускает системные проблемы (например, жалобы на скорость обслуживания повторяются, но не воспринимаются серьёзно из-за разрозненности данных).

**Решение:** AI-агент, который за 10–15 секунд суммирует любое количество отзывов, выявляет паттерны и даёт конкретные рекомендации.

## 2. Маппинг на четыре потока IT4IT

### S2P — Strategy to Portfolio (Стратегия → Портфель)

**Бизнес-проблема:** Малый ресторанный бизнес не может позволить себе команду аналитиков для обработки обратной связи от клиентов. Отзывы на Google Maps, 2GIS, Instagram читаются вручную и бессистемно.

**Почему AI — правильное решение:**
- LLM (Large Language Model) способна обрабатывать неструктурированный текст на любом языке
- Стоимость API (Google Gemini Free Tier) — 0 тенге для малого объёма запросов
- ROI: замена 2 часов ручного труда в день на 15 секунд автоматического анализа
- Масштабируемость: инструмент одинаково хорошо работает с 5 и с 500 отзывами

**Стратегическое решение:** Включить инструмент в операционный процесс SME как еженедельный/ежедневный ритуал анализа репутации.

### R2D — Requirement to Deploy (Требования → Развёртывание)

**Архитектурные решения, принятые в ходе разработки:**

**Выбор технологий:**
- **Frontend:** Streamlit — позволяет создать рабочий веб-интерфейс без знания HTML/CSS/JS. Идеален для B2B инструментов внутреннего использования.
- **AI Backend:** Google Gemini 2.0 Flash — быстрая и бесплатная модель, достаточная для текстового анализа. Альтернатива GPT-4 без необходимости платной подписки.
- **Деплой:** Локальный запуск через `streamlit run app.py`. В production — возможен деплой на Streamlit Cloud (бесплатно).

**Архитектурные промпты к AI агенту (Session Log):**

> *Prompt 1:* "Спроектируй Streamlit приложение для анализа клиентских отзывов с использованием Google Gemini API. Интерфейс на русском языке. Пользователь вставляет текст отзывов, нажимает кнопку, получает структурированный отчёт."

> *Prompt 2:* "Добавь в боковую панель ввод API ключа. Ключ не должен отображаться в открытом виде (type='password'). Конфигурация модели должна происходить только после нажатия кнопки анализа."

> *Prompt 3:* "Добавь D2C компонент: счётчик токенов через model.count_tokens(), секцию FinOps с информацией о лимитах бесплатного тира Gemini."

> *Prompt 4:* "Исправь: убрать неиспользуемый import os, обеспечить корректную обработку исключений при ошибке API."

**Итерации:** Архитект (студент) направлял агента от общего к конкретному: сначала структура приложения, затем UX детали, затем мониторинг, затем баг-фикс.

### R2F — Request to Fulfill (Запрос → Выполнение)

**Как SME потребляет этот инструмент:**

1. Менеджер/владелец открывает браузер: `http://localhost:8501`
2. Вводит API ключ в боковую панель (один раз, хранится в сессии)
3. Копирует отзывы из Google Maps / 2GIS / Instagram в текстовое поле
4. Нажимает кнопку **«Анализировать»**
5. Получает отчёт: тональность, топ-3 похвалы, топ-3 жалобы, 3 рекомендации
6. Видит сколько токенов использовано (FinOps прозрачность)

**Пользовательский сценарий (User Journey):**
- Каждое утро менеджер тратит 3 минуты: вставил отзывы за вчера → получил сводку → поставил задачу команде.
- Вместо 2 часов чтения — 3 минуты осмысленного действия.

### D2C — Detect to Correct (Обнаружение → Исправление)

**Мониторинг и контроль качества:**

| Риск | Метод обнаружения | Исправление |
|------|-------------------|-------------|
| Галлюцинации AI | Промпт с чёткой структурой вывода. Пользователь видит исходные отзывы рядом с анализом | Перезапустить анализ, уточнить промпт |
| Превышение лимитов API | Счётчик токенов в UI, сообщение об ошибке 429 | Уменьшить объём отзывов, подождать сброса лимита |
| Ошибка сети / API | try/except блок с понятным сообщением на русском | Пользователь видит инструкцию по действию |
| Неверный API ключ | Валидация входных данных + обработка ошибки аутентификации | Сообщение об ошибке с подсказкой |

**FinOps:**
- Google Gemini Free Tier: 1,500 запросов/день, 1M токенов/день
- Для SME с ~50 отзывами/день: расход ~5,000 токенов/день → полностью в рамках бесплатного тира
- В production: подключить Google Cloud Billing Alerts при превышении $5/месяц

## 3. Рефлексия: управление AI агентом вместо написания кода

### Что было неожиданным

Самым сложным оказалось не техническое задание, а **точность формулировок**. Когда говоришь агенту "добавь мониторинг" — он может добавить логирование в файл, дашборд, метрики или просто print(). Нужно было говорить: "добавь счётчик токенов через метод count_tokens() и покажи его в UI через st.metric()".

### Архитектурные узкие места

1. **Границы ответственности:** Сложно было объяснить агенту, что API ключ не должен быть захардкожен, а должен вводиться пользователем в runtime. Агент по умолчанию предлагал читать из переменных окружения — рабочий, но неудобный для SME вариант.

2. **UX vs функциональность:** Агент хорошо пишет логику, но без явного указания делает интерфейс "для разработчика", а не "для владельца кафе". Пришлось явно указывать: "добавь подсказки, упрости язык, убери технические термины из UI".

3. **D2C как afterthought:** Мониторинг и обработка ошибок не были заложены в первый промпт. Это типичная ошибка — сначала строишь happy path, потом добавляешь resilience. В реальном проекте нужно требовать D2C компоненты с первой итерации.

### Вывод

Роль Product Architect в связке с AI агентом — это работа переводчика между бизнес-требованиями и техническими решениями. Агент обладает техническими знаниями, но не понимает контекст бизнеса. Архитект знает контекст, но делегирует реализацию. Ценность создаётся именно на стыке — в качестве системных промптов и архитектурных решений, которые направляют агента.
